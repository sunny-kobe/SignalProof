# 信号

## 原始信号

用户要求严肃完整运行 SignalProof，而不是 lite 初筛，研究个人 AI 内容商业项目到底应该做什么方向；不能预设为中文 AI coding / 独立开发者，也不能回到 AI coding 工作流诊断服务。

## 为什么值得进入流程

- 这是一个方向选择问题，不是单个内容选题问题。
- 过早选定“中文 AI coding / 独立开发者”会把项目锁在用户熟悉领域，但不一定是最好的商业入口。
- 过早选定“泛 AI 内容号”会落入 AI 工具搬运、AI 新闻、AI 内容农场和低信任内容竞争。
- 当前要回答的是：哪个细分人群、痛点和内容资产组合，最适合用 Codex、SignalProof、HyperFrames / html-video 跑出可验证反馈。

## 初步用户

本 case 比较四类候选用户：

1. 中文 AI 实践型创作者、知识型自媒体和小团队：需要把海外 AI 信号转成可信中文内容资产。
2. 产品、运营、市场和 B2B 内容团队：需要理解 AI search / GEO / AI visibility 变化，并把报告转成实际内容动作。
3. AI coding / Agent / GitHub 项目关注者：需要实测、避坑和技术判断，但不一定是最大商业入口。
4. 泛 AI 工具学习者和普通职场人：流量面更宽，但付费意愿、差异化和信任门槛更弱。

## 当前边界

- 本 case 只判断“应该做什么方向”和“下一步怎么验证”，不声称市场已验证。
- 真实反馈目前为空；没有发布、账号数据、私信、下载或付费意向。
- HyperFrames / html-video 是内部生产引擎，不是第一产品。
- 不做 Web App、SaaS dashboard、workflow builder 或通用知识库。
- 不把 AI coding 工作流诊断/搭建服务作为默认方向。
- weak / blocked 证据只能支持继续研究或低成本内部实验；medium 证据最多支持 7 天公开或半公开验证。
