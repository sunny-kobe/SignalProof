# 信号

## 原始信号

用户要求外部反馈和批量分发先暂停，先精简 SignalProof 的前几步，尤其是研究类流程：研究是否真的确定、是否准确、有没有办法验证，以及哪些 API 或网站需要用户本人认证。

## 为什么值得进入流程

- 这条信号直接指向当前流程最薄的位置：研究证据质量。
- 如果研究阶段不能判断来源覆盖、交叉验证和反证，后续验证计划会建立在弱判断上。
- 本 case 不做外部反馈，只把研究准确性质量门和授权缺口固化为仓库规则。

## 初步用户

- 高频使用 Codex / Claude Code / MCP / 本地 AI coding 工作流的人。
- 想把机会判断沉淀为可复用个人资产的人。
- 当前用户本人是第一使用者和验收者。

## 当前边界

- 不声称市场验证。
