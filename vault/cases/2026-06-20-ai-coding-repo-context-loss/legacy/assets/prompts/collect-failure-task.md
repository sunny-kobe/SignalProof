# 私聊收集真实失败任务 Prompt

我最近在验证一个很窄的 AI coding 工作流：

不是教人写 `AGENTS.md`，而是审计 repo context 能不能减少 Codex / Claude Code 在真实项目里写偏。

我想找 5 个高频 AI coding 用户做一个小测试：你给我一个最近 AI 写偏的任务，我用 checklist 帮你脱敏跑一次 before/after。

我不会把它包装成成功案例，也不会公开你的 repo。

我只想验证这件事到底有没有用：一份更瘦、更贴近失败模式的 `AGENTS.md` / `CLAUDE.md`，是否能让 agent 少犯同类错。

你最近有没有一个 AI 明显写偏的任务？

## 需要的信息

- 你用的工具：Codex / Claude Code / Cursor / 其他。
- 任务描述。
- AI 写偏在哪里。
- 当前是否有 `AGENTS.md` / `CLAUDE.md`。
- repo 是否可以脱敏描述。
- 你最希望 agent 遵守什么规则。
