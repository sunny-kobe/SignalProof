# 5 个目标用户私聊台账

## 使用方式

把你准备私聊的人填进表格。只记录必要信息，不记录敏感隐私。

| 编号 | 人群类型 | 渠道 | 是否已发送 | 是否回复 | 反馈等级 | 是否提交真实任务 | 下一步 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | Codex 高频用户 |  | no | no |  | no |  |
| 2 | Claude Code 高频用户 |  | no | no |  | no |  |
| 3 | Cursor / AI coding 高频用户 |  | no | no |  | no |  |
| 4 | 小团队工程负责人 |  | no | no |  | no |  |
| 5 | 熟悉 repo context 的工程师 |  | no | no |  | no |  |

## 私聊文案

```text
我最近在验证一个很窄的 AI coding 工作流：

不是教人写 AGENTS.md，而是审计 repo context 能不能减少 Codex / Claude Code 在真实项目里写偏。

我想找 5 个高频 AI coding 用户做一个小测试：
你给我一个最近 AI 写偏的任务，我用 checklist 帮你脱敏跑一次 before/after。

GitHub repo 在这里：
https://github.com/Lan-Suny/repo-ai-context-audit

目前只有 synthetic demo，不是成功案例。
我不会公开你的 repo，也不会把它包装成成功案例。

你最近有没有一个 AI 明显写偏的任务？
```

## 反馈分级

- A：提交真实任务 + 具体写偏点 + 愿意看 before/after。
- B：没有任务，但给出具体反对意见或改进建议。
- C：收藏、点赞、泛泛支持。
- D：无关讨论。
