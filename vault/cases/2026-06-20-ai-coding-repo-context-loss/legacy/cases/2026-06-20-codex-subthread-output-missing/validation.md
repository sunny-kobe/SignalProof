# 验证计划

## 验证对象

子线程文件级验收是否能防止 SignalProof 误把“线程完成”当成“产物完成”。

## 验证方式

用刚发生的 Day 2 审计子线程作为案例：

1. 检查子线程是否完成。
2. 检查目标文件是否存在。
3. 发现目标文件缺失。
4. 源线程补齐文件。
5. 把失败写入台账和 flow review。
6. 反补 skill。

## 反馈来源

内部流程反馈，不是外部用户反馈。

## 成功标准

- 文件缺失被发现；
- 没有误写成 completed；
- 源线程补齐；
- 规则沉淀到 skill；
- 后续脚本能检查类似问题。

## Kill 条件

- 只记录问题，不形成规则；
- 只靠人工记忆，不做台账；
- 未来再出现同类问题却无法定位。

## 当前结果

验证通过。该失败已经转成协议规则。
