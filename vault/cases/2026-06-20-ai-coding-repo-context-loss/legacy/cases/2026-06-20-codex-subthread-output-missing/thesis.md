# 机会判断

## 当前判断

```text
继续，作为 SignalProof Protocol 的质量门沉淀
```

## 最小切口

子线程文件级验收门。

## 第一批使用场景

- 跨线程反方辩论；
- 子线程调研；
- 子线程产物方案；
- 子线程流程 QA；
- 子线程代码实现或验收。

## 不做什么

- 不做独立线程管理产品；
- 不做 UI dashboard；
- 不做自动 agent 平台。

## 成功标准

- 子线程任务台账里每个输出路径都能被检查；
- 缺文件时不会误标 completed；
- 源线程能接管修复并记录流程教训；
- 规则写进 skill 和脚本。

## 放弃条件

- 如果实际使用中这个检查从不发现问题；
- 如果系统工具已经原生保证产物存在和结构化回执；
- 如果检查成本明显大于收益。
