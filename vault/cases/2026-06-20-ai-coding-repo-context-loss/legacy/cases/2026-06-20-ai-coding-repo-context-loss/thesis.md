# 机会判断

## 当前判断

收窄后继续。

这个机会不能被定义成“AI coding 上下文平台”或“自动 repo 理解系统”。真正值得验证的是：一个 AI coding power user 是否愿意用一套轻量审计流程，改善真实 repo 中 agent 写偏的问题。

## 是否继续

- 继续 / 收窄 / 暂停 / 放弃：收窄后继续。

## 第一批用户

- Codex / Claude Code 高频用户。
- 有真实业务 repo 和反复 AI 写偏经历的人。
- 小团队中负责推动 AI coding 工作流的人。

## 最小切口

- 一份 repo context audit checklist。
- 一份瘦身版 `AGENTS.md` / `CLAUDE.md` 模板。
- 一个 before/after demo。
- 一个收集真实失败任务的 72 小时验证动作。

## 不做清单

- 不做平台。
- 不做 dashboard。
- 不做自动扫描器。
- 不做 SaaS。
- 不做插件市场。
- 不承诺“让 AI 永远理解项目”。
- 不把 synthetic demo 包装成真实案例。

## 成功标准

- 72 小时内至少 3 个具体使用信号，例如收藏、复制模板、提交真实失败任务、提出具体问题。
- 至少 1 个真实失败任务进入 before/after。
- 至少 1 条高质量反对意见让 checklist 变窄或变清楚。

## 放弃条件

- 只有泛泛点赞，没有具体使用意图。
- 没人愿意提交真实失败任务。
- 真实 before/after 没有可观察改善。
- 目标用户认为官方 `AGENTS.md` / `CLAUDE.md` 文档已经足够。

## 下一步行动

- 等待子线程的反方、产物方案和流程 QA 回执。
- 合并后生成 validation、artifact、feedback、decision、asset、flow-review 和 report。
