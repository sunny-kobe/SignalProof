# 判断

## 当前结论

推荐插件已经完成第一轮安装和试跑，但不应全部默认使用。SignalProof 的插件策略应该从“插件覆盖最大化”改成“证据增益最大化”。

## 最小切口

每个 case 开始时先选择最小必要插件：

1. 调研是否需要公开信号、私有资料、论文、SEO/流量或最近 30 天讨论。
2. 验证是否需要页面、GUI、运行态、表格或文档产物。
3. 反馈是否已有真实数据源。
4. 资产化是否需要录制、表格、文档、PPT 或 HTML 可视化。

## 不做清单

- 不为了“覆盖所有插件”而调用无数据源插件。
- 不把 `blocked/auth-or-tool` 当作 0 结果。
- 不把 Browser 失败或 Computer Use 权限问题写成流程不可用。
- 不因为装了产品分析插件就提前做 SaaS 化判断。

## 成功标准

- 默认插件能在无额外授权时产生可复核证据。
- 条件插件有明确触发条件。
- 失败插件有降级路线。
- 产物能帮助用户复盘，而不是只增加文件数量。

## 放弃条件

- 某插件连续多个真实 case 都不能改变判断。
- 某插件需要高维护成本但只产生弱证据。
- 某插件让流程变慢，却没有减少不确定性。
