# 信号

## 原始信号

用户确认当前不应预设为中文 AI coding / 独立开发者方向，需要先研究个人 AI 内容商业项目到底应该选择什么细分方向。

## 为什么值得进入流程

- 如果过早限定为“中文 AI coding / 独立开发者”，会把内容商业项目锁死在用户熟悉但不一定最有市场反馈的窄人群里。
- 当前真正要验证的不是某个内容选题，而是“哪类人群 + 哪类痛点 + 哪类内容资产”最值得进入 30 天账号实验。
- 本 case 是 `lite` 方向选择，不代表市场验证完成。

## 初步用户

- 候选用户 1：中文 AI 内容创作者、知识型自媒体、公众号/小红书运营者。
- 候选用户 2：产品、运营、市场、小团队负责人，需要把海外 AI 信号转成业务判断或内容资产。
- 候选用户 3：AI coding / Agent / GitHub 项目关注者，包括开发者、独立开发者和 AI 工作者。
- 候选用户 4：泛 AI 工具学习者和普通职场人。

本轮不预设最终人群，只比较候选方向的可验证性、付费可能、用户匹配度和用户个人能力优势。

## 当前边界

- 不声称市场验证。
- 不默认产品化。
- 不做 SaaS 或 dashboard。
- weak / blocked 证据只能支持继续研究或低成本内部实验。
- HyperFrames / html-video 是内部生产引擎，不是第一产品。
- 不回到 AI coding 工作流诊断服务默认方向。
