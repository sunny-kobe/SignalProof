# AI 内容商业项目启动包 v0

生成日期：2026-06-22

来源 case：`vault/cases/2026-06-22-个人-ai-内容商业方向完整验证`

证据等级：medium。这个包只支持 7 天低成本实验和 30 天账号验证，不支持直接产品化、服务化或 SaaS。

## 0. 项目方向

工作代号：海外 AI 信号实验室

一句话定位：

> 帮中文 AI 实践型创作者、知识型自媒体和小团队，把海外 AI 信号筛成可信判断、可发布内容和可复用资料包。

更短的外部表达：

> 海外 AI 信号到中文内容资产。

## 1. 第一阶段目标

第一阶段不是做品牌、课程、SaaS、自动化工作台，也不是证明长期商业模式成立。

第一阶段只验证 3 件事：

1. 你能否稳定选出值得中文用户关注的海外 AI 信号。
2. 你能否把信号转成有判断、有来源、有反方、有行动清单的内容样例。
3. 真实用户是否表现出复用意图：收藏、评论、私信、下载、关注、问模板、问后续更新或问服务。

## 2. 目标用户

优先级从高到低：

| 优先级 | 用户 | 他们要解决的问题 | 不要误判成 |
| --- | --- | --- | --- |
| P0 | 中文 AI 实践型创作者、知识型自媒体、小团队 | 看不完海外信息，不知道哪些值得转成内容/流程/资料包 | 泛 AI 学习者 |
| P1 | 产品、运营、市场、B2B 内容团队 | AI search / GEO / AI visibility 改变分发入口，但不知道怎么转成实际动作 | 传统 SEO 教程用户 |
| P2 | AI 实践者、开发者、独立开发者 | 想知道 Agent / GitHub / AI coding 项目是否值得试、怎么试、坑在哪里 | 只看热闹的开发者 |

第一阶段主用户是 P0，P1 是最接近付费痛点的主题线，P2 是你的技术信任来源。

## 3. 不做什么

- 不做 AI coding 工作流诊断服务。
- 不把 AI coding / Agent / GitHub 实测作为唯一定位。
- 不做泛 AI 新闻号。
- 不做泛 AI 工具搬运号。
- 不做全文翻译、转录稿搬运或付费资料改写。
- 不做 YouTube 摘要/翻译工具。
- 不做 Web App、SaaS dashboard、workflow builder 或通用知识库。
- 不承诺“自动化批量生产爆款内容”。

## 4. 第一批栏目

| 栏目 | 目标用户 | 内容承诺 | 可沉淀资产 |
| --- | --- | --- | --- |
| AI search / GEO / AI visibility | 产品、运营、市场、内容团队 | 把 AI 搜索和可见性变化转成内容动作 | AI visibility 行动清单、内容审计表、案例卡 |
| AI creator workflow | 知识型自媒体、小团队、创作者 | 把 AI 工具和海外案例转成可复用内容流程 | 选题评分表、内容生产 SOP、资料包模板 |
| AI coding / Agent / GitHub 实测 | AI 实践者、开发者、独立开发者 | 判断项目是否值得试、怎么试、坑在哪里 | 实测清单、避坑卡、工作流模板 |

栏目占比建议：

- `AI search / GEO / AI visibility`：40%
- `AI creator workflow`：40%
- `AI coding / Agent / GitHub 实测`：20%

这样不会锁死在开发者小圈子，也不会失去你的技术可信度。

## 5. 7 天实验

实验目标：产出 3 条可发布内容样例和 1 个免费资料包样例，并尽量获得真实反馈。

| 天数 | 动作 | 产物 | 通过标准 |
| --- | --- | --- | --- |
| Day 0 | 建新项目结构，确定主平台和记录表 | `direction.md`、`metrics/feedback-ledger.md` | 项目能开始记录信号和反馈 |
| Day 1 | 选 5 个海外 AI 信号 | 5 张 signal card | 每个信号有来源、目标用户、反方和评分 |
| Day 2 | 跑 5 个 lite 判断 | 5 个 mini proof | 至少 3 个信号值得转内容 |
| Day 3 | 生成第 1 条内容样例 | 小红书图文或公众号短笔记 | 有一手来源、反方、行动清单 |
| Day 4 | 生成第 2 条内容样例 | 小红书图文或公众号短笔记 | 能复用同一模板 |
| Day 5 | 生成第 3 条内容样例 | 60 秒脚本或长笔记 | 能覆盖第三条栏目线 |
| Day 6 | 组装免费资料包样例 | 《海外 AI 信号到中文内容资产样例包》 | 包含 3 个信号卡、3 个内容样例、评分表、复盘表 |
| Day 7 | 发布或半公开收集反馈 | 发布记录/反馈记录 | 如果未发布，只能记为内部产物验证 |

7 天结束只做一个决策：

```text
继续 / 收窄 / 暂停
```

不要在第 7 天直接做付费产品，除非已经出现明确私信、下载、复用请求或付费咨询。

## 6. 30 天计划

| 周期 | 目标 | 动作 | 决策点 |
| --- | --- | --- | --- |
| 第 1 周 | 验证内容资产样例是否成立 | 5 个信号、3 条内容、1 个免费资料包 | 是否有人收藏、下载、评论或私信 |
| 第 2 周 | 验证模板是否可复用 | 围绕表现最好的 2 个栏目重复 6-8 条 | 哪个栏目有更强复用意图 |
| 第 3 周 | 验证低价资料包或模板包 | 只在有真实需求时做 19-99 元小测试 | 是否有人愿意为省时间/拿模板付费 |
| 第 4 周 | 决策继续/收窄/暂停 | 写公开或内部复盘，整理资产库 | 是否进入订阅、资料包、服务或继续免费验证 |

## 7. 信号来源

优先来源：

- 官方博客、release note、产品文档、论文、报告。
- GitHub repo、issue、PR、discussion。
- Hacker News、Reddit、X、YouTube 评论、Newsletter。
- QuestMobile、Stack Overflow、a16z、CMI、IAB 等公开报告。

第一轮可从本次 case 已命中的信号池继续：

1. Marketers are asking how to turn AI visibility reports into actual work.
2. Clients are asking for AI detection reports with content delivery.
3. Google summarize-this-video style features may damage creator traffic.
4. AI search / GEO tracking tools are being packaged quickly by tool vendors.
5. AI slop is lowering trust in generated content.

这些只是候选信号，进入内容前仍要逐条跑来源核验和反方判断。

## 8. 信号评分表

每项 0-2 分，总分 16 分。

| 维度 | 0 分 | 1 分 | 2 分 |
| --- | --- | --- | --- |
| 一手来源 | 只有二手中文转述 | 有英文二手来源 | 有官方/原帖/仓库/报告 |
| 目标用户明确 | 泛 AI 用户 | 大致是创作者/团队 | 明确到某类工作流用户 |
| 中文信息差 | 中文已大量同质化 | 有部分信息差 | 中文几乎没有高质量解释 |
| 行动价值 | 只能看热闹 | 能形成观点 | 能形成清单、模板或流程 |
| 可验证性 | 无法测试 | 可查证但难实测 | 能试跑、截图、对比或复现 |
| 反方强度 | 没有反方 | 有风险但不清晰 | 有明确限制和不适用人群 |
| 资产化潜力 | 一次性资讯 | 可转成内容卡 | 可转成模板/资料包/SOP |
| 合规风险 | 高且不清楚 | 中等可控 | 低风险且可标注来源 |

决策阈值：

- 12-16 分：优先转内容样例。
- 9-11 分：保留，先补研究或换角度。
- 0-8 分：放弃或只存入信号池。

## 9. 内容样例模板

每条内容必须包含：

```text
1. 一句话判断
2. 来源和可信度
3. 它和中文用户有什么关系
4. 我验证了什么
5. 反方和坑
6. 适合谁 / 不适合谁
7. 可执行清单
8. 可复用资产
9. 行动引导
```

小红书图文结构：

```text
封面：一个具体问题或反常识判断
第 1 张：一句话结论
第 2 张：信号来自哪里
第 3 张：为什么和中文用户有关
第 4 张：我怎么验证/拆解
第 5 张：适合谁
第 6 张：坑和反方
第 7 张：可执行清单
第 8 张：评论关键词 / 领取资料 / 看完整笔记
```

公众号长笔记结构：

```markdown
# 标题
## 一句话结论
## 来源与可信度
## 背景
## 核心拆解
## 实测或证据
## 对中文用户的价值
## 可执行清单
## 风险、版权与 AI 标识说明
## 原始链接
```

60 秒短视频脚本结构：

```text
0-3 秒：具体问题或反常识结论
3-10 秒：来源和可信度
10-35 秒：三点拆解
35-50 秒：判断、适合谁、不适合谁
50-60 秒：行动引导
```

## 10. 免费资料包样例

标题：

> 海外 AI 信号到中文内容资产样例包

内容：

1. 3 张海外 AI 信号卡。
2. 3 条内容样例。
3. 1 张信号评分表。
4. 1 张发布复盘表。
5. 1 份来源和合规说明。

资料包定位是反馈入口，不是付费产品。它要验证用户是否愿意下载、收藏、转发、提问或要求后续更新。

## 13. Codex 执行协议

用户需要拍板：

- 是否公开发布。
- 发布到哪个账号和平台。
- 是否使用真实账号登录态。
- 是否把某条反馈当成强信号。
- 是否进入付费测试。

Codex 应主动执行：

- 找信号、列来源、保存原始链接。
- 做研究、反方、评分和内容初稿。
- 生成小红书/公众号/短视频脚本不同版本。
- 生成免费资料包样例。
- 维护反馈表和发布记录。
- 在 7 天结束时给出继续/收窄/暂停建议。

## 14. 继续和止损标准

7 天继续标准：

- 至少 3 条内容样例完成。
- 至少 1 个资料包样例完成。
- 如果已发布，出现收藏、评论、私信、下载、关注或具体问题中的任意有效反馈。
- 如果未发布，只能判断内部生产流程是否可复用，不能判断市场需求。

30 天继续标准：

- 至少 20 条内容资产。
- 至少 2 个栏目模板明显比其他模板更容易获得收藏、评论、私信或下载。
- 出现用户明确问资料包、模板、后续更新、服务或付费。
- 单条内容生产成本下降，而不是每条都重新研究。

止损标准：

- 只有浏览，没有收藏、评论、私信、下载或关注。
- 只能靠夸张标题、搬运或侵权素材获得数据。
- 用户只觉得“资讯好看”，没有复用、下载或付费线索。
- 生产成本过高，SignalProof 和 HyperFrames / html-video 没有实际降本。

## 15. 立即开工任务

新项目创建后，第一轮直接做这 5 件事：

1. 建目录和 3 个账本：`direction.md`、`metrics/feedback-ledger.md`、`metrics/publishing-ledger.md`。
2. 从候选信号池选 5 个信号，按评分表打分。
3. 选最高分的 3 个信号，各生成 1 个内容样例。
4. 组装免费资料包 v0。
5. 由用户拍板是否发布；发布后 72 小时记录真实反馈。

## 16. 当前结论

这个方向值得开始，但只能按实验推进。

最重要的判断：

- 主方向是“海外 AI 信号到中文内容资产”。
- AI coding / Agent / GitHub 是可信栏目，不是主定位。
- HyperFrames / html-video 是内部生产引擎，不是对外产品。
- 第一个商业信号不是收入，而是复用意图。
- 没有真实发布和反馈前，所有结论都停留在 direction validated for experiment。
